# Settings.ini File in nbdev

The `settings.ini` file is a configuration file in nbdev projects that stores project metadata and settings. It's typically located at the repository root and contains:

- Library and notebook paths
- Documentation output path
- GitHub repository details
- Version information
- Project description
- Author and license information
- Other project-specific metadata

The `version` field in `settings.ini` follows semantic versioning, which is a common versioning scheme that uses three numbers separated by dots: `MAJOR.MINOR.PATCH`. Here's what each part typically represents:
- `MAJOR`: Incremented for incompatible API changes or major new features. For example, `1.0.0` would indicate a stable public API.
- `MINOR`: Incremented for backward-compatible new features. For example, `0.1.0` would indicate the first feature/release in initial development.
- `PATCH`: Incremented for backward-compatible bug fixes. For example, `0.0.1` would indicate a tiny patch.

# the pyproject.toml file 
this file is used to specify the project dependencies and build system requirements. It is a standard file in Python projects that use modern packaging tools. In the context of nbdev, it also includes a `[tool.nbdev]` section that specifies nbdev-specific settings, such as the paths for notebooks and documentation.


# How nbdev Works in This Repository

nbdev is a notebook-first development system. Notebooks in `nbs/` are the source of truth; Python modules in `mini_rlm/` and documentation are **generated** from them — never edited directly.

## Repository Layout

```
mini-rlm/
├── nbs/                          # Source notebooks (edit these)
│   └── rlm_lisette_v2.ipynb
├── mini_rlm/                     # Auto-generated Python package (do not hand-edit)
│   ├── __init__.py
│   ├── _modidx.py                # Auto-generated by nbdev, maps symbols to modules
│   └── rlm_lisette_v2.py         # Exported from nbs/rlm_lisette_v2.ipynb
├── pyproject.toml                # [tool.nbdev] section points nbs_path and doc_path
└── Makefile                      # Convenience targets wrapping nbdev CLI commands
```

`pyproject.toml` registers `mini_rlm._modidx` as the nbdev entry point and sets:
- `nbs_path = 'nbs'` — where notebooks live
- `doc_path = 'docs'` — where the generated documentation site is written

## The `#|export` Directive

Any notebook cell that starts with `#|export` is exported to the corresponding Python module when you run `nbdev-export`. Cells without that directive are documentation, exploration, or tests only.

## Core Development Workflow

```
Edit nbs/*.ipynb  →  make nbdev-export  →  make nbdev-test  →  make nbdev-clean
```

| Makefile target    | nbdev command         | What it does                                              |
|--------------------|-----------------------|-----------------------------------------------------------|
| `nbdev-export`     | `nbdev_export`        | Converts `#\|export` cells into `mini_rlm/*.py` modules  |
| `nbdev-test`       | `nbdev_test`          | Runs all notebook cells as tests                         |
| `nbdev-clean`      | `nbdev_clean`         | Strips cell outputs and metadata to keep git diffs small |
| `nbdev-prepare`    | `nbdev_prepare`       | Runs export + test + clean in one step (use before push) |
| `nbdev-preview`    | `nbdev_preview`       | Serves the documentation site locally via Quarto         |

**Before every push**, run:
```bash
make nbdev-prepare
```

## Git Hooks

`make nbdev-hooks` installs a pre-commit hook (`nbdev_install_hooks`) that automatically cleans notebook metadata on every commit. This prevents irrelevant JSON noise in diffs caused by cell execution counts and output hashes.

Run once after cloning:
```bash
make nbdev-hooks
```

## First-Time Setup

```bash
make install          # create .venv and install requirements
make nbdev-install    # install nbdev into .venv
make nbdev-hooks      # install git pre-commit hooks
make nbdev-export     # generate mini_rlm/ from notebooks
```

## Publishing

| Makefile target  | What it does                                     |
|------------------|--------------------------------------------------|
| `nbdev-pypi`     | Builds and uploads the package to PyPI           |
| `nbdev-conda`    | Builds and uploads to Conda                      |
| `nbdev-release`  | Runs both PyPI and Conda release in sequence     |

## What Not to Edit by Hand

- `mini_rlm/*.py` — regenerated by `nbdev-export`; hand-edits are overwritten
- `mini_rlm/_modidx.py` — regenerated automatically; maps module symbols for nbdev
- `docs/` — regenerated by `nbdev-preview` / `nbdev-docs` via Quarto
