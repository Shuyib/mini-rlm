# AGENTS.md — mini-rlm Agent Guide

This file is for AI coding agents (Claude Code, Codex, Copilot, Cursor, Hermes, etc.) working with the mini-rlm repository. It covers project structure, conventions, and how to navigate the codebase effectively.

## Project Overview

mini-rlm is a lightweight Python toolkit for **Recursive Language Modeling** — giving an LLM the ability to step back, chunk up a big context, and call a helper LLM on each piece before answering. Built on [lisette](https://github.com/fastai/lisette), [litellm](https://github.com/BerriAI/litellm), and [toolslm](https://github.com/fastai/toolslm).

## Golden Rule: nbdev — Notebooks Are the Source

**Do NOT edit files in `mini_rlm/` directly.** They are auto-generated from `nbs/rlm_lisette_v2.ipynb`.

Workflow:
1. Edit the notebook at `nbs/rlm_lisette_v2.ipynb`
2. Run `make nbdev-export` to regenerate `mini_rlm/` modules
3. Run `make nbdev-test` to verify
4. Run `make nbdev-prepare` before pushing (export + test + clean)

If you must make a quick edit to the generated .py file for testing, note that it WILL be overwritten on next export.

## Key API Surface

All public interfaces live in `mini_rlm.rlm_lisette_v2`:

| Function | Purpose |
|---|---|
| `rlm(query, context)` | Drop-in: question + big text → answer |
| `RLM(context)` | Class: more control, chat history, configurable models |
| `prep_shell(context)` | Set up Python REPL with context + llm_query |
| `llm_query(query)` | Sub-LLM call from inside the REPL |
| `run_repl(code)` | Execute Python in the REPL |
| `generate_massive_context()` | Test data with hidden "magic number" |
| `print_history(rlm)` | Pretty-print conversation with emoji markers |
| `mm(graph)` | Render Mermaid diagrams (notebook) |
| `tool_mermaid(graph)` | Return Mermaid HTML blocks |

## How RLM Works

1. User passes a query + context to `rlm()` or `RLM()`
2. A Python REPL is created with `context` and `llm_query()` available
3. The root model gets a system prompt explaining available tools
4. The model writes Python code to explore, chunk, delegate, and collect
5. Code runs in the REPL — output comes back to the model
6. When ready, the model answers the original query

The system prompt includes concrete chunking strategies (by character count, by markdown headers, maintaining buffers).

## RLM Animation

The `rlm-animation/` directory contains a visual explainer rendered with **cairo** (pycairo) and encoded with ffmpeg. The animation covers the full RLM pipeline in 5 phases:

| Phase | Time (normal) | Content |
|---|---|---|
| Problem | 0.0-1.0s | Document scroll overflowing context window |
| Define + Chunk | 1.0-2.0s | "RLM: Recursive Language Model" → doc splits into 5 chunks |
| Delegate | 2.0-3.3s | Chunks → arrows → sub-LLM circles + code snippet |
| Aggregate | 3.3-4.2s | Sub-LLM summaries → yellow arrows → ROOT LLM |
| Answer | 4.2-5.0s | FINAL ANSWER box → "pip install mini-rlm" CTA |

To re-render:
```bash
cd rlm-animation && python3 render_v3.py
```

Requires: `pycairo`, `ffmpeg`.

The renderer uses a cohesive 4-color palette:
- BLUE (#58C4DD) — ROOT LLM, key concepts
- GREEN (#83C167) — sub-LLMs, processing
- YELLOW (#FFD600) — final answer, highlights
- DIM (#666666) — document, context

## Common Agent Tasks

### Adding a feature
1. Open `nbs/rlm_lisette_v2.ipynb`
2. Add your code in a cell with `#|export` flag
3. Run `make nbdev-export`
4. Test: `make nbdev-test`
5. Run `make nbdev-prepare` before committing

### Debugging
- Check `rlm-animation/hermes_notes.md` for session history
- The model uses OpenRouter by default — check `OPENROUTER_API_KEY` is set
- RLM context is loaded via `prep_shell()` — inspect that function for REPL setup issues
- Sub-LLM queries go through `llm_query()` which wraps litellm

### Creating or modifying the animation
In `rlm-animation/render_v3.py`, each phase is a `t < time` block in `render_frame()`. The script is 551 lines of pure cairo — no external scene framework. Key helpers: `draw_text()`, `draw_arrow()`, `fill_circle()`, `rounded_rect()`. Easing is cubic smoothstep via `ease(t)`.

## Style & Conventions

- Python 3.8+ compatible
- nbdev-first: notebooks with `#|export` cell flags
- Ruff for linting
- Docstrings follow numpy-style
- Type hints encouraged but not enforced
- The mini_rlm package follows fastai conventions (lisette, toolslm)
